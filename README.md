# Система учёта заявок сотрудников

## Описание проекта
Проект представляет собой консольное приложение для учёта заявок сотрудников, разработанное на Python с использованием SQLite в качестве базы данных. Система позволяет управлять сотрудниками, создавать и отслеживать заявки, а также формировать различные отчёты.

## Бизнес-процесс
Заявка проходит следующие статусы:
1. **Новая**
2. **В работе**
3. **Выполнена**

### Правила перехода статусов
Переходы между статусами строго последовательны:
- `Новая` → `В работе`
- `В работе` → `Выполнена`

**Запрещены:**
- Прямой переход `Новая` → `Выполнена`.
- Любой откат к предыдущему статусу.

### Переназначение исполнителя
Смена исполнителя разрешена для заявок в статусах `Новая` и `В работе`. Для заявок в статусе `Выполнена` переназначение невозможно.

## Функциональные требования
1.  **Сотрудники:**
    -   Учёт ФИО, подразделения и должности.
2.  **Заявки:**
    -   Создание заявок с полями: номер, дата создания, автор, исполнитель, описание, срок выполнения, статус.
3.  **Работа с заявками:**
    -   Изменение статуса заявки с проверкой допустимости перехода.
    -   Изменение исполнителя.
    -   Вывод списка заявок с фильтрацией по статусу, исполнителю, подразделению, а также просроченных заявок.
4.  **Отчётность:**
    -   Количество заявок по каждому статусу.
    -   Количество просроченных заявок.
    -   Количество выполненных заявок по исполнителям.

## Технические требования
-   **Язык программирования:** Python 3.11+.
-   **СУБД:** SQLite (файл `tracker.db`).
-   **Драйвер БД:** Встроенный модуль `sqlite3` (без ORM).
-   **Объектная модель:** Реализована объектная модель предметной области.
-   **Архитектура:** Код структурирован и легко расширяем (слои: `domain`, `db`, `repositories`, `services`, `cli`).
-   **Безопасность:** Все SQL-запросы используют параметризацию для предотвращения SQL-инъекций.

## Инструкция по запуску
1.  **Клонируйте репозиторий:**
    ```bash
    git clone <URL_репозитория>
    cd request_tracker
    ```
2.  **Установите зависимости:**
    ```bash
    pip install -r requirements.txt
    ```
3.  **Запустите приложение:**
    ```bash
    python main.py
    ```
    При первом запуске `main.py` будет инициализирована база данных `tracker.db` и заполнены начальные статусы и правила переходов.

## Генерация тестовых данных
Для генерации большого объема тестовых данных (1000+ сотрудников, 1 000 000+ заявок) выполните:
```bash
python scripts/generate_test_data.py
```

## Замер производительности и оптимизация
Для замера производительности целевого запроса и демонстрации оптимизации с помощью индексов выполните:
```bash
python scripts/benchmark_query.py
```
Скрипт выведет план выполнения запроса до и после создания составного индекса, а также время выполнения, демонстрируя ускорение.

### Обоснование оптимизации
Без индекса SQLite вынужден выполнять полное сканирование таблицы (`SCAN`) и сортировать результаты отдельно. С составным индексом `requests(executor_id, status_id, due_date)` планировщик запросов может использовать `Index Scan`, что позволяет быстро найти нужные записи и получить их уже отсортированными, значительно сокращая время выполнения запроса.
